spec-debug

2020-03-05

Debug helper which I’ve been always missing - minimal helper for instrumenting (clojure.spec.alpha/valid?) with more verbose output.

This should be treated as debug facility and not be enabled on production by default - especially, when there are spec/valid? statements expected to fail non-catastrophically used in performance-sensitive parts of code.

Kudos to @github/bhb for expound library which made this little helper possible.

My setup

Following profiles.clj configuration:

$ cat .lein/profiles.clj
{:user {:dependencies [[org.clojure/tools.trace "0.7.10"]
                       [otwieracz/spec-debug "0.1.0"]]
        :injections [(require '[clojure.tools.trace :refer [trace-vars
                                                            untrace-vars]])
                     (require '[otwieracz.spec-debug :refer [enable-spec-debug!
                                                             disable-spec-debug!]])]
        :repl-options {:timeout 360000}
        :aot :all}}

Enables use of enable-spec-debug! and trace-vars from tools.trace in every Leiningen project, drastically improving debugging experience - no more guesswork.

See spec-debug on GitHub for more info.
